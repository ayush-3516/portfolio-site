Building a multi-provider fallback chain that actually fails over

Why fallback matters

Every LLM provider has a bad day. Rate limits spike, a region degrades, a model gets deprecated mid-quarter. If your AI feature is wired to a single provider, your product inherits all of that risk directly. On Meraki — an AI sales-intelligence platform serving 100+ reps — call-coaching reports had to generate reliably or the whole workflow stalled.

The answer was a provider-agnostic orchestration layer: Anthropic Claude first, Gemini second, OpenAI third, with caching and observability wrapped around all three.

The problem with a single provider

A naive integration treats the provider SDK as infrastructure. But a 429 or a 503 shouldn't surface to the user — it should trigger a transparent retry on the next provider. The trick is normalizing request and response shapes so the rest of the system never knows which model answered.

Designing the chain

Each provider implements the same interface. The orchestrator walks the chain in priority order and returns the first success:

orchestrator.py
CHAIN = [Claude, Gemini, OpenAI]

async def generate(prompt, **kw):
    for model in CHAIN:
        try:
            return await model.complete(
                prompt, cache_ttl="24h", **kw)
        except (RateLimit, ServerError):
            log.warn(f"{model.name} failed, falling over")
            continue
    raise AllProvidersFailed()

Prompt caching for ~95% savings

Coaching reports reuse a large, stable system prompt. With 24-hour prompt caching, the static prefix is billed once and reused across the day's generations — which cut cost on cached calls by roughly 95%. The cache key is derived from the prompt prefix hash, so cache correctness is automatic.

Handling partial failures

The hard cases aren't clean errors — they're a provider that returns a 200 with truncated JSON, or streams half a response then drops. The orchestrator validates the shape of each response before accepting it, treating a malformed success the same as a hard failure and falling over.

Observability

Every run persists which provider answered, latency, token counts, cache hit/miss, and any fallover reason. That audit trail is what turns "the AI feels slow today" into a chart you can act on.

What I'd change next time

I'd add per-provider circuit breakers so a consistently-failing provider is skipped for a cooldown window instead of being retried on every request. Cheap to add, and it shaves tail latency on bad days.

THE PROBLEM

A legacy single-owner lead model couldn't support 100+ reps co-managing 50K+ leads. Ownership was binary, access was coarse, and there was no safe way to reassign work without race conditions or downtime.

ARCHITECTURE

A multi-owner lead system with an RBAC framework — primary, co-assigned, opportunity, and performer roles — backed by MongoDB sparse indexes for sub-millisecond access checks. A single-writer assignment service with idempotent backfill guaranteed correctness under concurrency.

CHALLENGES

Migrating live data with no downtime, normalizing 8+ CRM field variants with regex parsing, and keeping LLM costs sane at report-generation scale. Each demanded a different discipline — migration safety, data hygiene, and cost engineering.

SOLUTION

An 8-phase, feature-flagged rollout migrated the model with zero downtime. A multi-provider LLM fallback chain (Claude → Gemini → OpenAI) with 24h prompt caching powered AI call-coaching at ~95% lower cost on cached generations. Zoho + Merrito CRM integration ran via incremental ETL, and AI lead scoring drove automated multi-channel nurture across WhatsApp, LinkedIn, and email.

OUTCOME / IMPACT

Scaled to 100+ reps and 50K+ leads with a zero-downtime cutover and ~95% LLM cost reduction on cached reports — turning a brittle single-owner model into a system the sales org could grow on.

WHAT I LEARNED

Migrations are a product feature, not a chore. Feature-flagging every phase turned a terrifying cutover into a series of reversible, observable steps — and that confidence is what let us move fast without breaking the sales floor.
